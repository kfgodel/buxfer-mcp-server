# Shared Test Fixtures

Anonymized real Buxfer API responses used as the test contract for all three MCP server implementations (Kotlin, TypeScript, Python).

## Status

The files in `responses/` are **empty placeholders** (`{}`). They must be populated by running the capture tool before any tests can pass:

```bash
cd ../../api-recordings
export BUXFER_EMAIL="your@email.com"
export BUXFER_PASSWORD="yourpassword"
./run-capture.sh
```

See `../../api-recordings/CLAUDE.md` for the full workflow.

## Why shared fixtures?

- **No real API hits** during normal test runs — credentials not required.
- **Consistency** — all three implementations test against identical response shapes, catching any deserialization divergence.
- **Trustworthy** — fixtures are captured from a real Buxfer account then anonymized, so the actual API response format is preserved rather than guessed.

## WireMock mock server

Start the mock server (from `api-recordings/`):

```bash
docker compose up
```

WireMock listens on `http://localhost:8089`. Integration tests in all three language implementations should point their HTTP client at this address. The login endpoint returns `{"status":"OK","token":"test-mock-token"}` — use that token in test configuration.

## Using fixtures in Kotlin unit tests

`TestFixtureLoader.kt` in `kotlin/src/test/` loads fixtures by name. The path is injected via the `fixtures.dir` system property set in `build.gradle.kts`.

```kotlin
val json = TestFixtureLoader.load("accounts") // reads accounts.json
```

## Anonymization rules

Applied by the `.jq` transforms in `api-recordings/anonymize/`:

| Data type              | Replaced with                                  |
|------------------------|------------------------------------------------|
| Account names          | `"Test Account <id>"`                          |
| Bank names             | `"Test Bank"`                                  |
| Transaction descriptions | `"Transaction <id>"`                         |
| Contact names          | `"Contact 1"`, `"Contact 2"`, …               |
| Contact emails         | `"contact.1@example.com"`, …                   |
| Group names            | `"Test Group <id>"`                            |
| Group member names     | `"Member 1"`, `"Member 2"`, …                  |
| Loan entity names      | `"Test Entity 1"`, …                           |
| Reminder names         | `"Reminder <id>"`                              |
| Tag/budget/keyword names | kept as-is (generic categories, not personal) |
| IDs, amounts, dates    | kept as-is                                     |
